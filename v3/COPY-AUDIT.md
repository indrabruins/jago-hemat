## SECTION 1 — HERO

---

### 1A — Eyebrow Badge

**CURRENT:**
```
<span class="badge badge-orange">&#127942; Indonesia #1 Food Waste in ASEAN</span>
```
> Problem: "Indonesia #1 Food Waste in ASEAN" frames the brand with a negative superlative. Users don't self-identify with a country ranking. It reads like an award for a bad thing.

**RECOMMENDED — ID:**
```
<span class="badge badge-orange">&#127942; #1 Food Waste Solution di ASEAN</span>
```
**RECOMMENDED — EN:**
```
<span class="badge badge-orange">&#127942; #1 Food Waste Solution in ASEAN</span>
```

**WHY IT CONVERTS:** Leads with "solution" not "problem." Shifts brand from "the country that's bad at this" → "the country solving this." The ASEAN authority signal stays; the brand association flips positive. Competitors can't claim this positioning.

---

### 1B — Hero Headline (H1)

**CURRENT:**
```
Indonesia <span class="hl-y gradient-text">#1 Food Waste</span> di ASEAN.
Tapi <span class="hl-g">uangnya</span> untuk kamu.
```
> Problem: The first line is a stat, not a hook. Users read "Indonesia #1 Food Waste" → feel bad → look for an escape. The pivot to "but the money is for you" is clever but too subtle for a 3-second scan. The headline front-loads shame, not savings.

**RECOMMENDED — ID:**
```
Makanan <span class="hl-y gradient-text">Enak, 80% Lebih Murah</span>.
Selamatkan Bumi. Rinse.
```
**RECOMMENDED — EN:**
```
Delicious Food. <span class="hl-y gradient-text">Up to 80% Off.</span>
Save the Planet. Repeat.
```

**WHY IT CONVERTS:** TGTG's highest-converting hero copy leads with "save good food at great prices" — direct benefit, not country shame. Flashfood: "great food, great prices." The Jago Hemat name = "expert at saving" — this belongs in the headline. Lead with the reward (delicious + cheap), second with the mission. 3-second scan rule: what does the user **GAIN**?

### 1C — Hero Sub-headline

**CURRENT:**
```
14.7 juta ton makanan terbuang tiap tahun. 44% masih bisa dimakan.
Setiap pesanan Jago Hemat = makanan terselamatkan + dompet aman.
```
> Problem: The first sentence is a stat that belongs in a Reuters article, not a hero sub. It delays the value proposition. "Setiap pesanan Jago Hemat = ..." is good but buried. No urgency, no personal "you" language.

**RECOMMENDED — ID:**
```
Makanan berkualitas dari restoran favorit — sampai 80% lebih hemat.
Setiap pickup = makanan terselamatkan + dompet senang.
```
**RECOMMENDED — EN:**
```
Quality food from your favorite spots — up to 80% off.
Every pickup = food saved + money in your pocket.
```

**WHY IT CONVERTS:** Front-loads what the user GETS (quality food, 80% off, favorite restaurants). The waste stat lives in the How It Works or Impact section where it's contextual. TGTG sub-headlines lead with "great food at half price or less."

## SECTION 2 — HOW IT WORKS

---

```

```
> Problem: "3 menit dari buka app sampai hemat" — buried in the sub. "Cara Kerja" (How It Works) is a developer term, not a user benefit.

**RECOMMENDED — ID:**
```
<span class="section-tag">&#128203; Cuma 3 Langkah</span>

<h2 class="section-title">Start Saving in 3 Minutes</h2>
<p class="section-sub">Open app → pick your bag → save. No credit card, no hidden fees.</p>
```
**WHY IT CONVERTS:** "3 menit" moves to headline where it's more prominent. The arrow notation "Buka app → pilih → hemat" is a pattern interrupt — short, rhythmic, memorable. Adds trust signal (no credit card, no hidden fees) in the sub, directly addressing top signup friction.

### 2C — Step 2: "Beli Surprise Bag" Description

```

```
<div class="how-step-desc">Paket makanan premium dengan harga hingga 80% lebih murah. Isinya? Menu restoran di akhir hari — selalu berkualitas, selalu segar. Surprise, tapi kualitasnya pasti.</div>

**WHY IT CONVERTS:** "Isinya?" (what's inside?) followed by the answer is a micro-narrative — more engaging than a description. "Selalu berkualitas, selalu segar" removes the "will I get something gross?" fear. TGTG reframes Mystery Bags as delight, not risk.

### 3D — Urgent Overlay for Low-Time Cards

**CURRENT:** No urgent overlay on low-time deals. Cards with <15 min expiry display the same as 90-min cards.

**RECOMMENDED — add to JS card rendering for expMin < 15:**
```js
if (d.expMin < 15) {
  cardInnerHTML += '<div class="deal-urgent-overlay">&#128293; HABIS SEGERA — '+fmtTime(d.expMin)+'</div>';
}
```
**CSS needed:**
```css
.deal-urgent-overlay {
  position: absolute; top: 0; left: 0; right: 0;
  background: rgba(255,134,66,0.95);
  color: #fff; font-weight: 800; font-size: 13px;
  text-align: center; padding: 8px;
  border-radius: 12px 12px 0 0;
  animation: pulse 1.5s ease-in-out infinite;
}
@keyframes pulse { 0%,100%{opacity:1} 50%{opacity:0.7} }
```
**WHY IT CONVERTS:** Flashfood data shows urgency overlays ("X min left") on low-time deals drive 2-3x click-through. TGTG's most-engaged push notifications are expiry warnings. A "HABIS SEGERA" (ENDING SOON) overlay on <15-min cards creates a micro-flash-sale moment.

## APPENDIX B — MICROCOPY STANDARDS

### Button Copy Rules
1. **One action per button** — never combine two CTAs (download + incentive)
2. **Lead with the verb** — "Download Gratis", "Ajak Teman", "Daftar Sekarang"
3. **Specify the reward** — "3 Pickup Gratis" rather than "Gratis"
4. **Use action language** — "Share Link", "Ajak", "Gabung" vs passive "Terima"
5. **In EN toggle** — match TGTG/Flashfood convention: "Save Food. Save Money." / "Rescue a Bag"

### Badge Copy Rules
1. **Numbers + urgency** — "Cuma 3 lagi!" / "Only 3 left!"
2. **Specificity beats generality** — "127 paket di sekitarmu" > "500+ tersedia"
3. **Real-time language** — "Stok real-time" / " diperbarui otomatis"
4. **No negative framing in badges** — "Indonesia #1 Solution" beats "Indonesia #1 Waste"

### Timer Copy Rules
1. **Always label what the timer means** — "sisa waktu pickup" (remaining pickup time)
2. **Use natural language** — "2j 15m" not "02:15:00"
3. **For <15 min: trigger "HABIS SEGERA" overlay** — Flashfood data shows urgency overlays on low-time cards get 2-3x more clicks
4. **For >60 min: show in hours/minutes** — "3j 45m" not "225m"

### Form Error Copy Rules
1. **Be specific** — "Email tidak valid" not "Format salah"
2. **Be helpful** — "Contoh: nama@email.com" in placeholder
3. **On success: set expectation** — "Cek WhatsApp untuk follow-up"
4. **Error messages: use warning icon (&#9888;)** and orange/red color
5. **Success messages: use check icon (&#9989;)** and green color

### Social Proof Copy Rules
1. **Include savings amounts** in testimonials — "Hemat Rp 840K" > "great deal"
2. **Include location** — "Anisa, Jakarta" adds authenticity
3. **Include duration** — "3 bulan pakai" adds credibility
4. **For merchant testimonials** — lead with business impact, then mission
